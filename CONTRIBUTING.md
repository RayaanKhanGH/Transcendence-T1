# Contributing Guidelines

Thank you for considering contributing to **Transcendence T1**! We welcome contributions of all kinds – bug reports, feature requests, documentation improvements, and code contributions.

## How to Contribute

1. **Fork the repository**
2. **Create a new branch** for your work:
   ```bash
   git checkout -b feature/awesome-feature
   ```
3. **Make your changes** – ensure they follow the project's coding standards (see `docs/PROJECT_STANDARDS.md`).
4. **Run the test suite** to verify everything works:
   ```bash
   python tests/test_imports.py
   python tests/test_all_modules.py
   ```
5. **Commit your changes** with a clear, conventional commit message (see `docs/PROJECT_STANDARDS.md`).
6. **Push to your fork** and open a Pull Request.

## Pull Request Checklist
- [ ] Code follows the project's naming and style conventions.
- [ ] All new/changed code has appropriate docstrings and type hints.
- [ ] Tests have been added/updated and all pass.
- [ ] Documentation has been updated if applicable.
- [ ] The PR description clearly explains the changes.

## Reporting Issues
If you encounter a bug or have a suggestion, please open an issue using the provided templates in `.github/ISSUE_TEMPLATE/`.

## Code of Conduct
All participants are expected to adhere to the [Code of Conduct](CODE_OF_CONDUCT.md).
